# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Intelligence Layer — Matching and Outcome Surfacing.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> This is a React Native / Expo project (TypeScript). No shadcn, no web design system.
> All styling uses React Native `StyleSheet.create`. Design tokens are defined as explicit hex
> and numeric values derived from the existing codebase.

Source: derived from existing spacing in `ExpandableCard.tsx` and `MealLogScreen.tsx`.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Match row primary text | 14px | 400 (regular) | 1.4 |
| Insulin hint text | 12px | 400 (regular) | 1.3 |
| Section header "You've eaten this before" | 12px | 600 (semibold) | 1.2 |
| "Went well" indicator text | 11px | 600 (semibold) | 1.2 |

### Rationale

These four roles cover all new text in Phase 3. They slot beneath the existing type hierarchy:
- 16px semibold = meal name (existing `mealName` style)
- 13px regular = supporting metadata (existing `mealDate`, `carbEstimate`)
- 12px regular = secondary labels (existing `insulinBadgeText`, `statLabel`)

Match row primary text at 14px sits between metadata (13px) and meal name (16px) — readable
without competing with the meal name. All font weights are either 400 or 600 — the two weights
already in use across the app.

Source: existing type scale extracted from `ExpandableCard.tsx` and `MealLogScreen.tsx`.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#000000` | Page background (MealLogScreen, screen root) |
| Secondary (30%) | `#1C1C1E` | Card surfaces, match list container, inline autocomplete backdrop |
| Tertiary surface | `#2C2C2E` | Match row dividers, stats row background in expanded cards |
| Accent (10%) | `#30D158` | "Went well" green dot + text indicator ONLY |
| Destructive | `#FF3B30` | Not used in this phase |

### Accent reserved for

The green accent `#30D158` is reserved exclusively for the "Went well" indicator (one green dot +
"Went well" text) on match rows where `classifyOutcome()` returns `GREEN`. It must not appear on
any other element in Phase 3.

### Supporting semantic colors (existing — do not change)

| Value | Element |
|-------|---------|
| `#636366` | Muted text: insulin hint "(Xu last time)", section header, match row date |
| `#8E8E93` | Secondary labels and placeholder text |
| `#48484A` | TextInput placeholder text (existing) |
| `#0A84FF` | Blue interactive: tappable links (already used for refresh, "Change") |
| `#0A1A3A` | Insulin badge background (existing — reuse for insulin hint pill if used) |

### Glucose value colors (existing — reuse on match row peak value)

| Value | Condition |
|-------|-----------|
| `#FF3B30` | `mmol < 3.9` (low) |
| `#30D158` | `3.9 ≤ mmol ≤ 10.0` (in range) |
| `#FF9500` | `mmol > 10.0` (high) |

These are already implemented in `glucoseColor()` inside `ExpandableCard.tsx`. Phase 3 must
import and reuse this function — do not redeclare it.

Source: `ExpandableCard.tsx` StyleSheet + `OutcomeBadge.tsx` CONFIG map.

## Component Inventory

These are the exact components Phase 3 must produce or modify. Executor references this list.

### New: `MatchingSlot` (inline sub-component of `ExpandableCard`)

Replaces the Phase 2 placeholder at `ExpandableCard.tsx` lines 181–185.

**Trigger:** Rendered only when `matchData` is non-null and contains 2 or more matches.

**Layout:**
```
┌─────────────────────────────────────────┐
│ YOU'VE EATEN THIS BEFORE                │  ← 12px semibold, #636366, uppercase
├─────────────────────────────────────────┤
│ [Meal name] — Xu · [date] · peak X.X   │  ← 14px regular, #FFFFFF for name
│ [OutcomeBadge small]  ● Went well       │  ← badge left, green dot right (Green only)
├─────────────────────────────────────────┤
│ ... up to 5 rows                        │
└─────────────────────────────────────────┘
```

**Row layout:** Single `View` with `flexDirection: 'row'`, `alignItems: 'center'`, `gap: 8`.

**Row content (left to right):**
1. Meal name + em-dash + units — `Text` at 14px weight 400, `#FFFFFF`, `numberOfLines={1}`, `flex: 1`
2. Compact date — `Text` at 12px, `#636366`
3. Peak glucose value — `Text` at 12px, color from `glucoseColor(peak)` (reuse existing function)
4. `OutcomeBadge` at `size="small"` — reuse existing component
5. "Went well" indicator — `View` with green dot (`width: 6, height: 6, borderRadius: 3, backgroundColor: '#30D158'`) + `Text` "Went well" at 11px semibold `#30D158` — ONLY rendered when badge is `GREEN`

**Format string for row primary text:**
`{meal.name} — {session.insulinUnits}u`
Date shown on second line or inline after a `·` separator, formatted as: `"Wed 18 Mar"` using
`new Date(session.startedAt).toLocaleDateString('en-GB', { weekday: 'short', day: 'numeric', month: 'short' })`.

**Visibility rule:** If `matchData === null` or `matchData.matches.length < 2`, render nothing —
no placeholder, no empty state, complete silence. (CONTEXT.md D-04, D-14)

**Current session confidence warning:** If the card's own session has `confidence !== 'high'`
(i.e. 2+ meals were logged in the same 3-hour window), render a warning line **above** the
section header:
```
fontSize: 11, color: '#636366', fontStyle: 'italic'
```
Exact copy: `"Other meals were logged in this session — results may be affected"`
Rendered as a single `Text` element with `marginBottom: 4`. No icon.

**Per-row confidence warning on matched sessions:** If a matched historical session has
`confidence !== 'high'`, render a second line under the row's primary content:
```
fontSize: 11, color: '#636366', fontStyle: 'italic'
```
Exact copy: `"Other meals may have affected these results"`
Indented to align with the meal name (`paddingLeft: 0`, flows naturally beneath).

**Section divider:** `borderTopWidth: 1, borderTopColor: '#2C2C2E', paddingTop: 8` — matches
existing `matchingSlot` style already in `ExpandableCard.tsx` line 227. Do not change the
container style, only replace the interior.

### Modified: `MatchingSlotProps` in `src/components/types.ts`

Widen from `matchData: null` to:
```typescript
import type { MatchSummary } from '../services/matching';
export interface MatchingSlotProps {
  matchData: null | MatchSummary;
}
```
No other interface changes. `ExpandableCardProps.matchingSlot` field type flows through
automatically.

### New: Inline match list on `MealLogScreen`

**Position:** Directly below the meal name `TextInput`, before the insulin units label.

**Trigger:** `useEffect` watching `mealName` state. Fires after 300ms debounce when
`mealName.trim().length >= 2`. Hidden entirely when no matches or input length < 2. (CONTEXT.md D-06, D-07, D-10)

**Container style:**
```
backgroundColor: '#1C1C1E'
borderRadius: 12
marginBottom: 8
overflow: 'hidden'
```

**Row style:**
```
paddingHorizontal: 16
paddingVertical: 12    ← meets 44px minimum tap target
borderTopWidth: 1
borderTopColor: '#2C2C2E'   ← divider between rows (omit on first row)
```

**Row content:** Same layout as MatchingSlot rows above (meal name + units + date + peak + badge + "Went well" indicator).

**Tap behaviour:** Tapping a row:
1. Sets `mealName` field to the tapped match's meal name (auto-fill).
2. Does NOT pre-fill `insulinUnits`. (CONTEXT.md D-08, D-09, D-17)
3. Dismisses keyboard via `Keyboard.dismiss()`.
4. Hides the inline match list.

**Insulin hint:** After a match is tapped, show `(Xu last time)` adjacent to the "Insulin units"
label. Displayed as a `Text` element inline with the label row:
```
fontSize: 12, color: '#636366', fontStyle: 'italic'
```
This is display only. Never pre-fill the field. (CONTEXT.md D-09, D-17)

**Per-row confidence warning:** If a matched session has `confidence !== 'high'`, render a
second line under the row primary content using the same style as MatchingSlot:
```
fontSize: 11, color: '#636366', fontStyle: 'italic'
```
Exact copy: `"Other meals may have affected these results"`

**No results state:** List hidden entirely — no "no results" message. (CONTEXT.md D-10)

## Interaction Contract

| Interaction | Trigger | Response | Duration |
|-------------|---------|----------|----------|
| Live match search | `mealName` change (>= 2 chars) | Debounce 300ms → call `findSimilarSessions` | 300ms delay |
| Match row tap (MealLogScreen) | `onPress` on match row | Auto-fill name, show insulin hint, dismiss keyboard, hide list | Immediate |
| Matching slot visibility (ExpandableCard) | Card expand | Check match count >= 2; render or hide | Immediate (no animation) |
| "Went well" indicator | `classifyOutcome(match) === 'GREEN'` | Render green dot + "Went well" text | Immediate |

No new `LayoutAnimation` is introduced in this phase. The existing `easeInEaseOut` on card
expand/collapse is unchanged.

## Copywriting Contract

All copy is strictly historical. No forward-looking language. No advice. No recommendations.
Legal constraint from REQUIREMENTS.md and CONTEXT.md D-16.

| Element | Exact Copy |
|---------|------------|
| Matching section header (history cards) | `YOU'VE EATEN THIS BEFORE` |
| Match row format | `{Meal name} — {X}u · {Wed 18 Mar} · peak {X.X} mmol/L` |
| "Went well" indicator | `Went well` |
| Insulin hint (MealLogScreen, after tap) | `({X}u last time)` |
| No matches (hidden — no copy shown) | — |
| Current session confidence warning | `"Other meals were logged in this session — results may be affected"` |
| Matched row confidence warning | `"Other meals may have affected these results"` |
| Error state (match load failure) | Silent failure — hide slot entirely, no error shown to user |

### Forbidden language (non-negotiable)

Do not use any of these in Phase 3 UI strings:
- "you should", "try", "recommended", "suggest", "consider"
- "next time", "for this meal" (forward-looking framing)
- "Your usual dose", "typical dose" (implies recommendation)

Source: REQUIREMENTS.md PATT-01, PATT-02; CONTEXT.md D-16, D-13; CLAUDE.md safety rules.
